# Agentic RAG 完整指南

> **版本说明**
> - **v1** (`agentic_rag.py`)：基础版 - 查询改写、迭代检索、问题分解
> - **v2** (`agentic_rag_v2.py`)：增强版 - 网络搜索、多源融合、冲突处理

## 一、功能概述

Agentic RAG 是一个智能问答系统，具备以下核心能力：

| 功能 | 说明 |
|------|------|
| **LLM意图判断** | 使用大模型智能判断用户意图（闲聊/问答/追问） |
| **知识库检索** | 向量检索 + BM25 + Rerank 三重检索 |
| **智能改写** | 自动改写不准确的查询词 |
| **问题分解** | 复杂问题分解为多个子问题分别检索 |
| **网络搜索** | 知识库不足时自动搜索网络（可选） |
| **多源融合** | 智能整合知识库和网络信息 |
| **会话记忆** | 支持多轮对话，记住上下文 |

## 三、意图判断流程

### 3.1 三种意图类型

```
用户输入
    ↓
┌─────────────────────────────────────────────┐
│              LLM 意图判断                     │
└─────────────────────────────────────────────┘
    ↓
    ├── chat (闲聊)
    │   ├── 问候：你好、您好、hi
    │   ├── 感谢：谢谢、感谢
    │   ├── 情感：今天心情不好、你真棒
    │   └── 身份：你是谁、你能做什么
    │   └→ 直接调用大模型对话，不检索知识库
    │
    ├── query (知识问答)
    │   ├── 询问：XX是什么、怎么XX
    │   ├── 流程：XX流程、如何办理
    │   └── 信息：XX标准、XX规定
    │   └→ 检索知识库
    │
    └── follow_up (追问)
        ├── 代词：那XX呢、它怎么样
        ├── 继续：还有呢、然后呢
        └── 澄清：不是这个、换一个
        └→ 结合历史上下文改写查询后检索
```

### 3.2 意图判断 Prompt

```
你是一个意图识别助手。分析用户输入，判断用户意图。

【用户输入】
{query}

【对话历史】
{history}

【意图类型】
1. chat - 闲聊、问候、感谢、情感表达等，不需要检索知识库
2. query - 知识问答，需要检索知识库获取信息
3. follow_up - 追问，需要结合历史上下文理解

【判断标准】
- 问候、感谢、告别、情感表达 → chat
- 询问具体信息、流程、规定等 → query
- 代词指代（"那..."、"它..."）、追问 → follow_up
- 无法确定时默认 → query

输出JSON：{"intent": "类型", "reason": "理由", "search_query": "检索词"}
```

## 四、Agent 决策流程

### 4.1 决策类型

| 决策 | 触发条件 | 执行动作 |
|------|---------|---------|
| `answer` | 信息足够 | 生成最终答案 |
| `kb_search` | 需要检索知识库 | 执行向量+BM25检索 |
| `web_search` | 知识库不足/需要最新信息 | 网络搜索 |
| `rewrite` | 查询词不准确 | 改写后重新检索 |
| `decompose` | 问题包含多个子问题 | 分解后分别检索 |

### 4.2 决策流程图

```
开始检索
    ↓
检索知识库
    ↓
┌─────────────────────────────────────────────┐
│           Agent 判断信息是否足够              │
└─────────────────────────────────────────────┘
    ↓
    ├── 足够 → 生成答案 → 结束
    │
    ├── 不够准确 → 改写查询 → 重新检索
    │
    ├── 问题复杂 → 分解为子问题 → 分别检索 → 合并答案
    │
    └── 需要最新信息 → 网络搜索 → 融合信息
```

## 五、检索策略

### 5.1 混合检索

```python
# 向量检索（语义相似）
vector_results = collection.query(query_vector, top_k=15)

# BM25检索（关键词匹配）
bm25_results = bm25.search(query, top_k=15)

# RRF融合（合并结果）
fused = reciprocal_rank_fusion([vector_results, bm25_results])

# Rerank重排（精确排序）
final = reranker.rerank(query, fused, top_k=5)
```

### 5.2 RRF 融合算法

```
RRF分数 = Σ (权重 / (k + 排名位置))

文档A: 向量排名1 → 1/(60+1) = 0.0164
       BM25排名3 → 1/(60+3) = 0.0159
       总分 = 0.0323
```

```python

# 处理问题

print(f"意图: {result['intent']}")      # chat/query/follow_up
print(f"答案: {result['answer']}")
```

## 十二、最佳实践

### 12.1 何时使用 Agentic RAG

✅ **推荐使用**：
- 复杂问题需要多轮检索
- 用户表达模糊需要改写
- 需要区分闲聊和知识问答
- 需要多轮对话记忆

❌ **不推荐使用**：
- 简单明确的问题（用传统RAG更快）
- 对响应时间敏感的场景

### 12.2 性能优化

```python
# 减少迭代次数
rag = AgenticRAG(max_iterations=2)

# 禁用网络搜索
rag = AgenticRAG(enable_web_search=False)
```

### 12.3 调试技巧

```python
# 打印详细过程
result = rag.process("问题", verbose=True)

# 查看推理过程
for step in result['reasoning']:
    print(f"第{step['iteration']}轮: {step['decision']['action']}")
```

## 附加篇：Agentic RAG深入优化与工作机制（整合自研发笔记）

Agentic RAG优化指南：提升准确率与智能性

Agentic RAG（智能体检索增强生成）是传统 RAG 的进阶形态，它将 LLM 的智能决策能力引入检索流程，从 "被动执行预设流程" 升级为 "主动思考 - 检索 - 反思" 的闭环系统。通过针对性的优化，我们可以将系统的回答准确率从传统 RAG 的 60% 提升至 95% 以上，同时显著增强回答的相关性与智能性。

一、Agentic RAG 的核心架构

与传统 RAG 的线性 "检索 - 生成" 流程不同，Agentic RAG 构建了动态的决策闭环，核心组件包括：

整个系统围绕决策 - 检索 - 评估 - 优化的循环展开，让系统能够像人类研究员一样，根据实时反馈动态调整检索策略，从而解决传统 RAG 面对模糊查询、复杂推理时的痛点。

二、分阶段优化策略：从检索到推理的全链路提升

1. 检索前：优化查询质量，解决 "输入模糊" 问题

用户的原始查询往往存在口语化、模糊、歧义等问题，直接用于检索会导致召回结果偏差。我们需要通过以下手段优化查询：

（1）智能查询重写与语义转换

将用户的口语化表述自动转换为精准的检索术语，同时生成多个查询变体提升召回率。

- 核心逻辑：利用 LLM 分析用户意图，提取核心概念，规范专业术语，补充领域上下文。

- 实战案例：

  - 用户输入："租房子没到期房东想涨房租怎么办？"

  - 初始检索：仅匹配到 0.07 分的无关条文

  - 重写后查询："租赁合同 出租人 单方变更租金 法律效力"

  - 二次检索：置信度提升至 0.99，精准匹配到民法典相关条款

- Prompt 示例：

你是查询重写专家。优化用户的法律查询，提高检索效果。重写策略：
1. 提取核心法律概念和关键词
2. 规范法律术语表述
3. 补充相关法律领域信息
4. 生成多个查询变体以提高召回率

（2）复杂问题分解

对于需要多步推理的复杂问题（如多跳问答），自动将其拆解为多个子问题，逐个检索解决。
例如："2024 年某公司的营收增长率是多少？" 会被拆解为：

1. 该公司 2024 年的营收是多少？

2. 该公司 2023 年的营收是多少？

3. 计算两者的增长率

2. 检索中：提升召回精准度，过滤噪声

检索阶段是决定结果相关性的核心，我们需要通过多策略融合，确保召回最相关的内容：

（1）多路召回与融合排序

单一的向量检索或关键词检索都存在局限性，通过融合多种检索方式，覆盖不同的匹配场景：

- 向量检索：捕捉语义相似度，解决同义词、 paraphrase 问题

- 关键词检索（BM25）：捕捉精确匹配，解决专业术语、短查询的精准匹配

- 融合策略：使用 RRF（Reciprocal Rank Fusion）算法融合两种检索的排序结果，避免单一策略的偏差

- 动态权重调整：根据查询特征调整权重，短查询（<5 词）时提高 BM25 的权重，长查询时提高向量检索的权重

（2）基于 Reranker 的自信度决策门控

重排序模型不再只是排序工具，而是系统的路径决策器，通过置信度阈值过滤低质量结果：

- 技术实现：使用 BGE-Reranker-v2-m3 等交叉编码器模型，对初筛结果进行精排

- 阈值控制：实测设定 0.35 为动态判定阈值，若 Top-1 片段得分低于该值，判定当前知识不足，拦截生成请求，触发重写检索

- 效果：该机制可过滤 99% 的无关噪声，避免基于错误上下文生成幻觉回答

3. 检索后：质量评估与自我迭代

检索完成后，系统需要主动评估结果质量，不足则自动优化，形成闭环：

（1）多维度质量评估

从四个维度对检索结果进行量化评估，判断是否足够回答用户问题：

1. 相关性：检索结果与用户查询的匹配程度

2. 完整性：是否包含足够信息回答用户问题

3. 准确性：内容的准确性和权威性

4. 覆盖面：结果的多样性和全面性

- 阈值判断：总分≥32 分（四项各 10 分，80% 阈值）则判定为质量达标，否则触发补救流程

（2）推理反思与自我纠错

引入 \\Reasoning Reflection（推理反思）\\ 机制，让 Agent 主动检查自己的推理过程，找出未被验证的假设，针对性补充检索：

- 这是 Re2Search 智能体的核心创新，区别于传统的 ReAct 架构

- Agent 会先基于现有信息构建初步回答，然后反思其中哪些结论缺乏证据支持，基于这些缺失的信息生成新的检索查询

- 实验表明，该机制可将 F1 score 提升 10% 以上，尤其在复杂推理任务上效果显著

（3）分层补救机制

当本地知识库检索结果不达标时，自动触发外部搜索作为 fallback：

- 触发 SerpAPI 等网络搜索工具，从互联网获取补充信息

- 融合本地知识库和网络搜索的结果，生成综合性回答

- 这就是 Corrective RAG 的核心逻辑，解决了本地知识库覆盖不全、数据过时的问题

三、系统级优化：构建鲁棒的智能闭环

1. 避免 "循环检索" 陷阱

Agentic RAG 的闭环流程可能导致系统反复重写查询却无法获取有效结果，陷入无效循环，解决方案：

- 设置检索次数阈值：最多允许 3 次重写检索，超过后直接触发外部搜索或人工介入

- 置信度递增检查：每次检索后检查置信度是否有提升，若连续两次无提升则终止循环

2. 平衡智能性与效率

Agentic RAG 的多轮迭代会增加计算开销，通过以下手段优化成本：

- 轻量级决策模型：在决策、重写阶段使用小型 LLM（如 GPT-3.5、Qwen-7B），仅在最终生成阶段调用大模型

- 缓存机制：对高频查询的检索结果和决策路径进行缓存，相同问题直接复用结果，减少重复计算

- Token 优化：使用 TeaRAG 等框架，通过知识图谱压缩检索内容，减少推理过程的 Token 消耗，可减少 60% 以上的输出 Token

3. 安全与可解释性增强

- 证据溯源：在输出答案的同时，强制展示原始文档片段和置信度得分，让用户可以验证信息来源

- 思维链展示：展示 Agent 的内部推理过程，从口语问题到专业术语的转换、检索的迭代过程，提升系统的可解释性

- 安全护栏：加入输入过滤、输出审查、权限控制，避免敏感信息泄露和恶意输出

四、学术前沿：系统化的模型优化

除了工程层面的优化，最新的学术研究提供了更系统的优化框架：

1. RAG-Gym：三维度系统优化

RAG-Gym 是首个针对 Agentic RAG 的系统化优化平台，从三个维度全面提升性能：

1. 提示工程：构建 Re2Search 智能体，融合推理反思机制，超越传统的 ReAct 提示

2. 执行器调优：通过过程监督对 LLM 进行微调，实验表明 \\ 直接偏好优化（DPO）\\ 是最有效的算法，优于 SFT 和 PPO

3. 评判器训练：训练一个评判模型，在推理阶段自动筛选高质量的中间推理步骤，进一步提升结果质量

- 效果：将 ReAct agent 在 HotpotQA 上的 F1 score 从 41.09% 提升到 60.19%，比 Search-R1 等方法相对提升 3.2%~11.6%

2. 过程监督 vs 结果监督

传统的强化学习仅使用最终答案的正确性作为奖励（结果监督），而最新的 ReasonRAG 等研究证明：

- 细粒度的过程奖励能显著提升训练效率，仅需要 5k 训练样本就可以达到传统方法 90k 样本的效果

- 过程奖励可以对查询生成、证据提取、答案生成每个中间步骤进行监督，让 Agent 学会每一步的正确决策，而不只是最终结果

五、落地效果验证

在多个垂直领域的实践中，优化后的 Agentic RAG 展现了显著的效果提升：

- 医疗领域：在 ACL 损伤诊疗任务中，Agentic RAG 将 GPT-4 的准确率从 RAG 阶段的 94% 进一步提升到 95%，相比原生 LLM 的 58% 提升了 37 个百分点

- 法律领域：在民法典咨询任务中，Agentic RAG 将复杂问题的回答准确率从 62% 提升到 91%，解决了口语化查询匹配难的问题

- 网络安全领域：CyberRAG 框架通过 Agentic RAG 实现了 94.92% 的攻击分类准确率，同时将误报率降低了 40%

六、总结

提升 Agentic RAG 的准确率和智能性，核心是将系统从 "被动的检索工具" 升级为 "主动的智能决策体"，通过：

1. 查询优化解决输入模糊的问题

2. 多策略检索提升召回的精准度

3. 闭环反思实现自我纠错与迭代

4. 系统优化平衡效果与成本

5. 模型微调从底层提升 Agent 的决策能力

通过这套组合优化策略，你可以构建出一个能够自主思考、动态调整、可靠准确的智能检索系统，真正实现 "像人类专家一样" 的问答体验。
