# Agentic RAG 完整指南

> **版本说明**
> - **v1** (`agentic_rag.py`)：基础版 - 查询改写、迭代检索、问题分解
> - **v2** (`agentic_rag_v2.py`)：增强版 - 网络搜索、多源融合、冲突处理

## 一、功能概述

Agentic RAG 是一个智能问答系统，具备以下核心能力：

| 功能 | 说明 |
|------|------|
| **LLM意图判断** | 使用大模型智能判断用户意图（闲聊/问答/追问） |
| **知识库检索** | 向量检索 + BM25 + Rerank 三重检索 |
| **智能改写** | 自动改写不准确的查询词 |
| **问题分解** | 复杂问题分解为多个子问题分别检索 |
| **网络搜索** | 知识库不足时自动搜索网络（可选） |
| **多源融合** | 智能整合知识库和网络信息 |
| **会话记忆** | 支持多轮对话，记住上下文 |

## 三、意图判断流程

### 3.1 三种意图类型

```
用户输入
    ↓
┌─────────────────────────────────────────────┐
│              LLM 意图判断                     │
└─────────────────────────────────────────────┘
    ↓
    ├── chat (闲聊)
    │   ├── 问候：你好、您好、hi
    │   ├── 感谢：谢谢、感谢
    │   ├── 情感：今天心情不好、你真棒
    │   └── 身份：你是谁、你能做什么
    │   └→ 直接调用大模型对话，不检索知识库
    │
    ├── query (知识问答)
    │   ├── 询问：XX是什么、怎么XX
    │   ├── 流程：XX流程、如何办理
    │   └── 信息：XX标准、XX规定
    │   └→ 检索知识库
    │
    └── follow_up (追问)
        ├── 代词：那XX呢、它怎么样
        ├── 继续：还有呢、然后呢
        └── 澄清：不是这个、换一个
        └→ 结合历史上下文改写查询后检索
```

### 3.2 意图判断 Prompt

```
你是一个意图识别助手。分析用户输入，判断用户意图。

【用户输入】
{query}

【对话历史】
{history}

【意图类型】
1. chat - 闲聊、问候、感谢、情感表达等，不需要检索知识库
2. query - 知识问答，需要检索知识库获取信息
3. follow_up - 追问，需要结合历史上下文理解

【判断标准】
- 问候、感谢、告别、情感表达 → chat
- 询问具体信息、流程、规定等 → query
- 代词指代（"那..."、"它..."）、追问 → follow_up
- 无法确定时默认 → query

输出JSON：{"intent": "类型", "reason": "理由", "search_query": "检索词"}
```

## 四、Agent 决策流程

### 4.1 决策类型

| 决策 | 触发条件 | 执行动作 |
|------|---------|---------|
| `answer` | 信息足够 | 生成最终答案 |
| `kb_search` | 需要检索知识库 | 执行向量+BM25检索 |
| `web_search` | 知识库不足/需要最新信息 | 网络搜索 |
| `rewrite` | 查询词不准确 | 改写后重新检索 |
| `decompose` | 问题包含多个子问题 | 分解后分别检索 |

### 4.2 决策流程图

```
开始检索
    ↓
检索知识库
    ↓
┌─────────────────────────────────────────────┐
│           Agent 判断信息是否足够              │
└─────────────────────────────────────────────┘
    ↓
    ├── 足够 → 生成答案 → 结束
    │
    ├── 不够准确 → 改写查询 → 重新检索
    │
    ├── 问题复杂 → 分解为子问题 → 分别检索 → 合并答案
    │
    └── 需要最新信息 → 网络搜索 → 融合信息
```

## 五、检索策略

### 5.1 混合检索

```python
# 向量检索（语义相似）
vector_results = collection.query(query_vector, top_k=15)

# BM25检索（关键词匹配）
bm25_results = bm25.search(query, top_k=15)

# RRF融合（合并结果）
fused = reciprocal_rank_fusion([vector_results, bm25_results])

# Rerank重排（精确排序）
final = reranker.rerank(query, fused, top_k=5)
```

### 5.2 RRF 融合算法

```
RRF分数 = Σ (权重 / (k + 排名位置))

文档A: 向量排名1 → 1/(60+1) = 0.0164
       BM25排名3 → 1/(60+3) = 0.0159
       总分 = 0.0323
```

## 六、使用方式

### 6.1 命令行使用

```bash
# 交互模式
python agentic_rag.py

# 单次问答
python agentic_rag.py "出差补助标准是什么"

# 对比模式（与传统RAG对比）
python agentic_rag.py --compare "出差补助标准"
```

### 6.2 代码调用

```python
from agentic_rag import AgenticRAG

# 初始化
rag = AgenticRAG(max_iterations=3, enable_web_search=True)

# 处理问题
result = rag.process("出差补助标准是什么？")

print(f"意图: {result['intent']}")      # chat/query/follow_up
print(f"答案: {result['answer']}")
print(f"迭代次数: {result['iterations']}")
print(f"来源: {result['sources']}")
```

### 6.3 带历史上下文

```python
# 多轮对话
history = [
    {"role": "user", "content": "出差补助标准是什么？"},
    {"role": "assistant", "content": "根据规定，出差补助包括..."}
]

result = rag.process("那请假呢？", history=history)
# 会自动理解"那"指的是出差相关的请假
```

### 6.4 API 调用

```bash
# 启动API服务
python rag_api_server.py

# 发送请求
curl -X POST http://localhost:5001/chat \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "user_001",
    "session_id": null,
    "message": "出差补助标准是什么？"
  }'
```

## 十二、最佳实践

### 12.1 何时使用 Agentic RAG

✅ **推荐使用**：
- 复杂问题需要多轮检索
- 用户表达模糊需要改写
- 需要区分闲聊和知识问答
- 需要多轮对话记忆

❌ **不推荐使用**：
- 简单明确的问题（用传统RAG更快）
- 对响应时间敏感的场景

### 12.2 性能优化

```python
# 减少迭代次数
rag = AgenticRAG(max_iterations=2)

# 禁用网络搜索
rag = AgenticRAG(enable_web_search=False)
```

### 12.3 调试技巧

```python
# 打印详细过程
result = rag.process("问题", verbose=True)

# 查看推理过程
for step in result['reasoning']:
    print(f"第{step['iteration']}轮: {step['decision']['action']}")
```
