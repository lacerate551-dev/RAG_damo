# 文档级权限控制 - 技术实现说明

## 一句话总结

**权限划分存储在向量库中，权限判断在查询时根据用户角色动态过滤。**

## 核心文件清单

### 1. 文档摄入阶段（写入权限标记）

| 文件 | 作用 | 关键代码 |
|------|------|----------|
| `rag_demo.py` | 根据目录自动分配 `security_level` | `load_documents()` 函数，检测目录名写入元数据 |
| `rag_demo.py` | 向量检索时过滤 | `search_knowledge()` 函数，`where={"security_level": {"$in": allowed_levels}}` |

### 2. 用户认证阶段（获取用户权限）

| 文件 | 作用 | 关键代码 |
|------|------|----------|
| `auth.py` | JWT 认证 + 角色权限表 | `role_permissions` 表定义每个角色可访问的级别 |
| `auth.py` | `get_user_permissions()` | 根据用户角色返回 `["public", "internal", ...]` |

### 3. API 请求阶段（权限传递）

| 文件 | 作用 | 关键代码 |
|------|------|----------|
| `rag_api_server.py` | 从 Token 提取用户角色 | `@require_auth` 装饰器，`request.current_user` |
| `rag_api_server.py` | 获取权限并传给检索函数 | `auth_manager.get_user_permissions(role)` |

### 3. 权限获取 - `auth.py`

**位置：第 265-278 行 `get_user_permissions()` 函数**

```python
def get_user_permissions(self, role: str) -> List[str]:
    """获取角色对应的可访问安全级别"""
    cursor.execute(
        "SELECT allowed_levels FROM role_permissions WHERE role = ?",
        (role,)
    )
    row = cursor.fetchone()
    return json.loads(row[0])  # 返回 ["public", "internal", ...]
```

### 4. 查询时权限过滤 - `rag_demo.py`

**位置：第 1207-1242 行 `search_knowledge()` 函数**

```python
def search_knowledge(query, top_k=5, allowed_levels=None):
    # 构建权限过滤条件
    where_filter = None
    if allowed_levels:
        where_filter = {"security_level": {"$in": allowed_levels}}

    # 向量检索时过滤
    vector_results = collection.query(
        query_embeddings=[query_vector],
        n_results=recall_k,
        where=where_filter  # ← ChromaDB 权限过滤
    )

    # BM25 后处理过滤
    if allowed_levels:
        allowed_set = set(allowed_levels)
        filtered = [i for i in top_indices
                    if bm25_metas[i].get('security_level', 'public') in allowed_set]
```

## 权限流转完整路径

```
用户登录
    ↓
auth.py: 生成 JWT Token (包含 user_id, role)
    ↓
API 请求携带 Token
    ↓
@require_auth 装饰器验证 Token，提取 role
    ↓
auth_manager.get_user_permissions(role) → ["public", "internal", ...]
    ↓
search_knowledge(query, allowed_levels=["public", "internal", ...])
    ↓
ChromaDB: WHERE security_level IN ("public", "internal")
    ↓
只返回用户有权限的文档
```

## 权限存储位置

| 存储位置 | 内容 | 文件 |
|---------|------|------|
| ChromaDB 向量库 | 每个文档 chunk 的 `security_level` 元数据 | `chroma_db/` |
| BM25 索引 | 每个文档的 `security_level` 元数据 | `bm25_index.pkl` |
| SQLite | 用户账号、角色、角色权限表 | `sessions.db` |

## 如何修改权限

### 修改角色权限
```python
# 在 auth.py 的 default_permissions 中修改
("user", ["public"])  # 普通用户只能看公开文档
```

### 修改文档权限
```bash
# 移动文档到对应目录
mv documents/财务报表.xlsx documents/confidential/

# 重建知识库
python rag_demo.py --rebuild
```

### 添加新的权限级别
1. 在 `security_levels` 字典中添加新级别
2. 在 `role_permissions` 表中配置哪些角色可访问
3. 创建对应的文档目录
