# 出题与批卷系统集成方案

## 概述

将 `exam_manager.py` 中的出题工作流和批卷工作流集成到 `rag_api_server.py`，提供 REST API 供前端调用。

---

## 需求确认

| 需求项 | 用户选择 |
|--------|----------|
| 审核粒度 | 两者都需要（整份试卷审核 + 逐题审核） |
| 题库管理 | 完整管理（增删改查、搜索题目） |
| 审核记录 | 不需要（简单通过/驳回） |

---

## API 接口设计

### 1. 出题相关 API

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/exam/generate` | 生成试卷（草稿状态） | 登录用户 |
| GET | `/exam/list` | 获取试卷列表 | 登录用户 |
| GET | `/exam/<exam_id>` | 获取试卷详情 | 登录用户 |
| PUT | `/exam/<exam_id>` | 更新试卷 | 登录用户 |
| DELETE | `/exam/<exam_id>` | 删除试卷 | 管理员 |
| POST | `/exam/<exam_id>/review` | 审核试卷（整体/逐题） | 管理员 |
| POST | `/exam/<exam_id>/approve` | 通过审核 | 管理员 |
| POST | `/exam/<exam_id>/reject` | 驳回试卷 | 管理员 |

### 2. 批卷相关 API

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/exam/<exam_id>/grade` | 批阅试卷 | 登录用户 |
| GET | `/report/<report_id>` | 获取批阅报告 | 登录用户 |
| GET | `/report/list` | 获取批阅报告列表 | 登录用户 |

### 3. 题库搜索 API

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/questions/search` | 搜索题目 | 登录用户 |

## API 详细设计

### 1. 生成试卷

```
POST /exam/generate
Authorization: Bearer <token>
Content-Type: application/json

Request:
{
  "topic": "Python基础知识",
  "choice_count": 5,
  "blank_count": 3,
  "short_answer_count": 2,
  "difficulty": 2,
  "choice_score": 2,
  "blank_score": 3
}

Response:
{
  "exam_id": "uuid",
  "status": "draft",
  "choice_questions": [
    {
      "id": 1,
      "content": "题目内容",
      "options": ["A. 选项1", "B. 选项2", "C. 选项3", "D. 选项4"],
      "answer": "B",
      "analysis": "答案解析",
      "knowledge_points": ["知识点1"],
      "difficulty": 2
    }
  ],
  "blank_questions": [...],
  "short_answer_questions": [...],
  "total_count": 10,
  "total_score": 25
}
```

### 2. 审核试卷（整体）

```
POST /exam/<exam_id>/review
Authorization: Bearer <token>

Request:
{
  "action": "approve"
}

Response:
{
  "success": true,
  "status": "approved"
}
```

### 3. 审核试卷（逐题）

```
POST /exam/<exam_id>/review
Authorization: Bearer <token>

Request:
{
  "action": "partial",
  "questions": [
    {"type": "choice", "id": 1, "approved": true},
    {"type": "choice", "id": 2, "approved": false, "edit": {"content": "修改后的题目..."}},
    {"type": "blank", "id": 1, "delete": true}
  ]
}

Response:
{
  "success": true,
  "approved_count": 1,
  "edited_count": 1,
  "deleted_count": 1
}
```

### 4. 批阅试卷

```
POST /exam/<exam_id>/grade
Authorization: Bearer <token>

Request:
{
  "student_name": "张三",
  "answers": {
    "choice_1": "A",
    "choice_2": "B",
    "blank_1": "答案内容",
    "short_answer_1": "简答题作答..."
  }
}

Response:
{
  "report_id": "uuid",
  "student_name": "张三",
  "total_score": 15,
  "max_score": 25,
  "score_rate": 60.0,
  "questions": [
    {
      "type": "choice",
      "id": 1,
      "content": "题目内容",
      "student_answer": "A",
      "correct_answer": "B",
      "score": 0,
      "max_score": 2,
      "correct": false
    }
  ]
}
```

### 5. 题库列表

```
GET /exam/list?status=approved&page=1&limit=20

Response:
{
  "exams": [
    {
      "exam_id": "uuid",
      "name": "Python基础测试",
      "status": "approved",
      "total_count": 10,
      "total_score": 100,
      "created_at": "2024-04-02T10:00:00",
      "created_by": "admin"
    }
  ],
  "total": 5,
  "page": 1
}
```

### 6. 搜索题目

```
GET /questions/search?keyword=Python&type=choice&difficulty=2

Response:
{
  "questions": [
    {
      "exam_id": "uuid",
      "exam_name": "Python基础测试",
      "type": "choice",
      "question": {...}
    }
  ],
  "total": 15
}
```

## 实现步骤

### 阶段一：后端 API（exam_api.py）

1. 创建 Flask 蓝图 `exam_api`
2. 实现出题相关接口：
   - `generate_exam()` - 调用 Dify 工作流生成试卷
   - `list_exams()` - 列出试卷
   - `get_exam()` - 获取试卷详情
   - `update_exam()` - 更新试卷
   - `delete_exam()` - 删除试卷
   - `review_exam()` - 审核试卷
   - `approve_exam()` - 通过审核
   - `reject_exam()` - 驳回试卷

3. 实现批卷相关接口：
   - `grade_exam()` - 批阅试卷
   - `get_report()` - 获取报告
   - `list_reports()` - 报告列表

4. 实现题库搜索接口：
   - `search_questions()` - 搜索题目

### 阶段二：修改 exam_manager.py

1. 添加 UUID 生成
2. 添加状态管理函数
3. 添加审核函数
4. 添加题库 CRUD 函数

### 阶段三：前端测试页面

1. 创建 `exam.html` 基础结构
2. 实现出题面板
3. 实现审核面板
4. 实现批卷面板
5. 添加样式

### 阶段四：集成测试

1. 测试出题流程
2. 测试审核流程
3. 测试批卷流程
4. 测试题库管理

## 验证方式

```bash
# 启动服务
python rag_api_server.py

# 测试生成试卷
curl -X POST http://localhost:5001/exam/generate \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"topic": "Python基础", "choice_count": 3}'

# 测试获取列表
curl http://localhost:5001/exam/list \
  -H "Authorization: Bearer <token>"

# 测试批卷
curl -X POST http://localhost:5001/exam/<exam_id>/grade \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"student_name": "测试", "answers": {"choice_1": "A"}}'
```

## 注意事项

1. **权限控制**：审核相关接口需要管理员权限
2. **Dify API 配置**：需要在 config.py 中配置 Dify API Key
3. **题库目录**：默认为 `./题库/`，确保有写入权限
4. **批阅报告目录**：默认为 `./批阅报告/`
