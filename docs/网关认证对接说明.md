# 网关认证对接说明

## 概述

API 服务已修改为从网关注入的 HTTP Header 中读取用户信息，不再处理本地登录认证。

## 网关需要注入的 Header

| Header 名称 | 必需 | 说明 | 示例 |
|------------|------|------|------|
| X-User-ID | 是 | 用户唯一标识 | user001 |
| X-User-Name | 否 | 用户名 | 张三 |
| X-User-Role | 否 | 用户角色 | admin/manager/user |
| X-User-Department | 否 | 部门 | 技术部 |

## 角色映射

后端角色会自动映射到本地角色：

| 后端角色 | 本地角色 |
|---------|---------|
| administrator | admin |
| admin | admin |
| manager | manager |
| user | user |
| normal | user |
| guest | user |

如需添加新的角色映射，请修改 `auth_gateway.py` 中的 `ROLE_MAPPING` 字典。

## 权限级别

| 角色 | 可访问的安全级别 |
|------|------------------|
| admin | public, internal, confidential, secret |
| manager | public, internal, confidential |
| user | public, internal |

## 开发测试

在开发环境中，可以设置环境变量来模拟网关认证：

```bash
# Windows
set DEV_MODE=true

# Linux/Mac
export DEV_MODE=true
```

然后使用 Header 模拟用户：

```bash
# 测试普通用户
curl -H "X-User-ID: test001" -H "X-User-Name: 测试用户" -H "X-User-Role: user" \
     http://localhost:5001/auth/me

# 测试管理员
curl -H "X-User-ID: admin001" -H "X-User-Name: 管理员" -H "X-User-Role: admin" \
     http://localhost:5001/stats
```

## API 接口

### 公开接口（无需认证）
- `GET /health` - 健康检查

### 需要认证的接口
所有其他接口都需要通过网关访问，网关会注入用户信息。

### 管理员专用接口
以下接口需要 `admin` 角色：
- `GET /stats` - 系统统计
- `POST /sync` - 手动触发同步
- `POST /sync/start` - 启动文件监控
- `POST /sync/stop` - 停止文件监控
- `POST /graph/build` - 重建知识图谱
- `GET /audit/logs` - 审计日志

## 已移除的接口

以下接口已移除（由后端网关处理）：
- `POST /auth/login` - 用户登录
- `POST /auth/register` - 用户注册
- `POST /auth/change-password` - 修改密码
- `GET /auth/users` - 用户列表
- `PUT /auth/users/<user_id>` - 更新用户
- `DELETE /auth/users/<user_id>` - 删除用户

## 保留的接口

- `GET /auth/me` - 获取当前用户信息（从 Header 读取）

## 错误响应

### 401 未认证
```json
{
  "error": "缺少用户信息",
  "message": "请通过网关访问，或设置 DEV_MODE=true 进行开发测试"
}
```

### 403 权限不足
```json
{
  "error": "权限不足",
  "message": "此接口需要以下角色之一: admin",
  "your_role": "user"
}
```

| 文件 | 变更 |
|------|------|
| auth_gateway.py | 新增，网关认证模块 |
| rag_api_server.py | 移除登录路由，使用网关认证 |
| test_gateway_auth.py | 新增，测试脚本 |

## 注意事项

1. **角色映射**：如果后端的角色名称与本地不同，需要修改 `auth_gateway.py` 中的 `ROLE_MAPPING`
2. **开发模式**：生产环境请确保不设置 `DEV_MODE=true`
3. **前端**：前端由其他团队负责，需要他们确保请求经过网关
