## 🔴 严重问题（12个）

### 1. SQL 注入漏洞
**文件**: `auth.py` 第 290-292 行
```python
cursor.execute(f"UPDATE users SET {', '.join(updates)} WHERE user_id = ?", params)
```
**问题**: 使用 f-string 动态构建 SQL 查询，虽然参数化了值部分，但字段名拼接存在潜在注入风险。
**建议**: 使用白名单验证允许更新的字段名。

### 2. 硬编码默认密码
**文件**: `auth.py` 第 106 行
```python
self._create_user_internal(cursor, "admin", "admin123", "admin", "系统管理部")
```
**问题**: 默认管理员密码硬编码在代码中，且密码过于简单。
**建议**: 首次运行时强制要求设置密码，或生成随机初始密码。

### 3. JWT 密钥硬编码
**文件**: `auth.py` 第 48-51 行
```python
self.jwt_secret = jwt_secret or os.environ.get(
    "JWT_SECRET",
    "dev-secret-change-in-production"
)
```
**问题**: 默认 JWT 密钥硬编码在代码中。
**建议**: 移除默认值，强制从环境变量或配置文件获取。

### 4. API Key 暴露风险
**文件**: `rag_api_server.py` 第 939 行
```python
print("  默认管理员: admin / admin123")
```
**问题**: 在启动日志中打印默认凭证。
**建议**: 移除此输出或仅在首次启动时显示一次。

### 5. Neo4j Cypher 注入风险
**文件**: `graph_manager.py` 第 180-183, 283-289 行
```python
query = f"""
{'MERGE' if merge else 'CREATE'} (n:{entity_type} {{name: $name}})
...
query = f"""
MATCH (a {{name: $head}})
MATCH (b {{name: $tail}})
{'MERGE' if merge else 'CREATE'} (a)-[r:{relation_type_safe}]->(b)
```
**问题**: `entity_type` 和 `relation_type_safe` 直接拼接到 Cypher 查询中，可能存在注入风险。
**建议**: 使用白名单验证实体类型和关系类型。

### 6. 敏感信息明文存储
**文件**: `session_manager.py`, `auth.py`
**问题**:
- 密码使用 SHA-256 + salt 存储，但算法较旧
- 数据库文件 `sessions.db` 没有加密保护
**建议**: 使用 bcrypt 或 argon2 进行密码哈希；考虑数据库加密。

### 7. 文件路径遍历风险
**文件**: `rag_demo.py` 第 313-315 行
```python
filepath = os.path.join(root, filename)
ext = os.path.splitext(filepath)[1].lower()
```
**问题**: 没有验证文件路径是否在预期目录内。
**建议**: 添加路径规范化和验证，确保文件在 `DOCUMENTS_PATH` 内。

### 8. 命令注入风险
**文件**: `exam_manager.py` 第 33-48 行
**问题**: 调用外部 API 时未对 `inputs` 参数进行充分验证和清理。
**建议**: 验证所有输入参数，限制特殊字符。

### 9. 缺少速率限制
**文件**: `rag_api_server.py`
**问题**: 所有 API 端点都没有速率限制，容易遭受暴力破解和 DoS 攻击。
**建议**: 添加 Flask-Limiter 或类似的速率限制中间件。

### 10. 不安全的文件操作
**文件**: `exam_manager.py` 第 171-184 行
```python
filepath = os.path.join(QUESTION_BANK_DIR, f"{name}.json")
with open(filepath, 'w', encoding='utf-8') as f:
    json.dump(exam, f, ensure_ascii=False, indent=2)
```
**问题**: 文件名没有清理，可能存在路径遍历风险。
**建议**: 验证并清理文件名，使用 UUID 作为文件名。

### 11. 缺少 HTTPS 强制
**文件**: `rag_api_server.py` 第 986 行
```python
app.run(host='0.0.0.0', port=5001, debug=True, threaded=True)
```
**问题**:
- 没有 SSL/TLS 配置
- 绑定 `0.0.0.0` 监听所有网络接口
- `debug=True` 在生产环境暴露敏感信息
**建议**: 生产环境禁用 debug，使用反向代理 (nginx) 处理 HTTPS。

### 12. CORS 配置过于宽松
**文件**: `rag_api_server.py` 第 59 行
```python
CORS(app)
```
**问题**: CORS 配置允许所有来源访问 API。
**建议**: 配置明确的允许来源列表。

## 🟠 高优先级问题（18个）

### 13. 异常处理不完善
**文件**: 多个文件
**示例**: `rag_demo.py` 第 533-535 行
```python
except Exception as e:
    print(f"      PDF解析错误 {filepath}: {e}")
```
**问题**: 捕获所有异常但只打印，没有适当的日志记录和错误追踪。
**建议**: 使用 logging 模块，记录完整的堆栈跟踪。

### 14. 资源泄漏风险
**文件**: `graph_manager.py`
```python
def connect(self) -> bool:
    ...
    self.driver = GraphDatabase.driver(...)
```
**问题**: Neo4j driver 没有使用上下文管理器，可能造成连接泄漏。
**建议**: 确保在所有代码路径上都调用 `close()`，或使用 `try-finally`。

### 15. 全局变量滥用
**文件**: `rag_demo.py`
```python
embedding_model = SentenceTransformer(EMBEDDING_MODEL_PATH)
chroma_client = chromadb.PersistentClient(path=CHROMA_DB_PATH)
collection = chroma_client.get_or_create_collection(...)
reranker = None
bm25_index = BM25Index()
llm_client = OpenAI(...)
```
**问题**: 大量全局变量使代码难以测试和维护。
**建议**: 使用依赖注入或单例模式封装。

### 16. 配置敏感信息暴露
**文件**: `config.py` (存在)
**问题**: 配置文件可能包含明文 API 密钥。
**建议**: 使用环境变量或密钥管理服务。

### 17. 缺少输入验证
**文件**: `exam_api.py`
```python
topic = data.get('topic', '')
```
**问题**: 没有验证输入长度、类型和格式。
**建议**: 使用 Pydantic 或 marshmallow 进行输入验证。

### 18. 密码复杂度要求不足
**文件**: `auth.py` 第 147-148 行
```python
if len(password) < 6:
    return False, "密码至少6位", None
```
**问题**: 仅要求密码长度 >= 6，没有复杂度要求。
**建议**: 添加密码复杂度要求（大小写、数字、特殊字符）。

### 19. 会话固定攻击风险
**文件**: `session_manager.py`
**问题**: 会话 ID 在登录后没有重新生成。
**建议**: 在用户登录后重新生成 session_id。

### 20. 缺少 CSRF 保护
**文件**: `rag_api_server.py`
**问题**: 没有 CSRF Token 保护。
**建议**: 添加 Flask-WTF CSRF 保护。

### 21. 日志敏感信息
**文件**: 多个文件
**示例**: `agentic_rag.py` 第 130 行
```python
print(f"[用户] {query}")
```
**问题**: 用户查询可能包含敏感信息，直接打印到日志。
**建议**: 对敏感数据进行脱敏处理。

### 22. 内存泄漏风险
**文件**: `session_manager.py`
**问题**: 消息历史没有自动清理机制，依赖 `cleanup_expired_sessions()` 手动调用。
**建议**: 添加定时任务自动清理过期数据。

### 23. 前端敏感信息存储
**文件**: `chat-ui/app.js` 第 9-10 行
```javascript
token: localStorage.getItem(TOKEN_KEY),
user: JSON.parse(localStorage.getItem(USER_KEY) || 'null'),
```
**问题**: Token 和用户信息存储在 localStorage，容易遭受 XSS 攻击。
**建议**: 使用 HttpOnly Cookie 存储 Token。

### 24. 缺少请求超时
**文件**: `agentic_rag.py` 第 764 行
```python
response = requests.post(url, headers=headers, data=payload, timeout=10)
```
**问题**: 虽然设置了超时，但其他地方如 `exam_manager.py` 没有。
**建议**: 所有 HTTP 请求都应设置合理超时。

### 25. 缺少数据库备份机制
**文件**: `session_manager.py`, `auth.py`, `audit_logger.py`
**问题**: 没有数据库备份和恢复机制。
**建议**: 实现定期备份策略。

### 26. 不安全的 JSON 解析
**文件**: `entity_extractor.py` 第 266 行
```python

```
**问题**: 没有验证 JSON 数据结构。
**建议**: 验证解析后的数据结构是否符合预期。

### 27. 缺少参数验证
**文件**: `graph_rag.py` 第 445 行

**问题**: 对外部输入没有验证数量限制。

### 28. 缺少并发控制

**问题**: 多用户同时访问可能导致资源竞争。
**建议**: 添加数据库连接池和并发控制。

### 29. 文件上传漏洞风险
**文件**: `rag_demo.py`
**问题**: 文件类型验证仅基于扩展名，可能被绕过。
**建议**: 添加文件内容类型验证（Magic Number）。

### 30. 缺少权限校验
**文件**: `exam_api.py` 第 310-362 行
