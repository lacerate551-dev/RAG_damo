## 二、概念速查

### 传统RAG vs Agentic RAG

| 特性 | 传统RAG | Agentic RAG |
|------|---------|-------------|
| 检索次数 | 1次 | 可多次迭代 |
| 查询处理 | 原样使用 | 可改写/分解 |
| 决策能力 | 无 | LLM自主决策 |
| 错误恢复 | 无 | 可换策略重试 |
| 复杂问题 | 处理困难 | 分解后分别处理 |

### 流程对比

```
【传统RAG】
问题 → 检索 → 生成答案

【Agentic RAG】
问题 → Agent思考 → 决策 → 执行 → 评估 → (循环或结束)
                    ↓
         ┌─────────┼─────────┬─────────┐
         ↓         ↓         ↓         ↓
      answer    rewrite   decompose  expand
      (回答)   (改写查询) (分解问题) (扩展关键词)
```

## 二、核心组件

### 1. Agent控制器

```python
class SimpleAgenticRAG:
    def __init__(self, max_iterations=3):
        self.max_iterations = 3      # 最大迭代次数
        self.client = OpenAI(...)    # LLM客户端
```

### 2. 主流程 process()

```python
def process(self, query: str) -> dict:
    """
    主处理流程

    返回:
        answer: 最终答案
        iterations: 迭代次数
        reasoning: 推理过程
        contexts: 检索到的上下文
    """
```

### 3. 决策函数 _think()

| 决策 | 触发条件 | 执行动作 |
|------|---------|---------|
| `answer` | 信息足够 | 生成最终答案 |
| `rewrite` | 查询词不准确 | 改写后重新检索 |
| `decompose` | 问题含多个子问题 | 分解后分别检索 |
| `expand` | 结果太少/不相关 | 扩展关键词检索 |

## 五、使用示例

### 命令行使用

```bash
# 交互模式
python agentic_rag.py

# 单次问答
python agentic_rag.py "你的问题"

# 对比模式（传统 vs Agentic）
python agentic_rag.py --compare "你的问题"
```

### 代码调用

```python
from agentic_rag import SimpleAgenticRAG

# 创建Agent
agent = SimpleAgenticRAG(max_iterations=3)

# 处理问题
result = agent.process("科研项目申报需要哪些材料？")

# 获取结果
print(result["answer"])       # 答案
print(result["iterations"])   # 迭代次数
print(result["reasoning"])    # 推理过程
```

## 七、关键参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `max_iterations` | 3 | 最大迭代次数，防止无限循环 |
| `top_k` | 5 | 每次检索返回的文档数量 |
| `temperature` | 0.3 | Agent决策时的温度（低温度=更确定） |
| `max_tokens` | 500 | Agent决策时的最大输出token |

## 十一、常见问题

### Q: Agentic RAG比传统RAG慢多少？

A: 单次检索时差不多，但可能需要多轮迭代：
- 简单问题：1轮，约3-5秒
- 中等问题：2轮，约6-10秒
- 复杂问题：3轮，约10-15秒

### Q: 什么时候用Agentic RAG？

A:
- ✅ 问题表述模糊
- ✅ 复杂多步骤问题
- ✅ 需要综合多个信息源
- ❌ 简单明确的问题（传统RAG足够）
- ❌ 对延迟敏感的场景

### Q: max_iterations设多少合适？

A:
- 2-3轮：适合大多数场景
- 4-5轮：复杂问题，但会增加延迟
- 超过5轮不推荐，收益递减
