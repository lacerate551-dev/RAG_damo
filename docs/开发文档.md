## 一、项目概述

### 1.1 项目定位

本项目是智能出题系统的**核心知识服务层**，为上层 Dify 工作流提供知识检索能力。系统通过 RAG（检索增强生成）技术，实现基于企业制度文档的智能问答，支持：

- **知识库问答**：基于向量检索 + BM25 + Rerank 的混合检索
- **图谱推理**：基于 Neo4j 的多跳关系查询
- **网络搜索**：实时信息获取（天气、新闻等）
- **Dify 集成**：HTTP API 供工作流调用

### 1.3 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| API 服务 | Flask + Flask-CORS | RESTful API，支持跨域 |
| 向量检索 | ChromaDB + BGE-base-zh | 本地向量数据库 + 嵌入模型 |
| 关键词检索 | BM25 + jieba | 中文分词 + 倒排索引 |
| 重排序 | BGE-reranker-base | CrossEncoder 精排 |
| 知识图谱 | Neo4j | 实体关系存储 |
| 大模型 | Qwen3.5 (通义千问) | 问答生成、实体提取 |
| 网络搜索 | Serper API | Google 搜索接口 |

## 四、Dify 工作流集成

### 4.1 集成架构

```
┌─────────────────────────────────────────────────────────────────┐
│                     Dify 工作流                                  │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────────────┐ │
│  │  开始节点   │───▶│  LLM 节点   │───▶│      结束节点       │ │
│  │  (用户问题) │    │  (出题/批阅)│    │    (输出结果)       │ │
│  └─────────────┘    └──────┬──────┘    └─────────────────────┘ │
│                            │                                    │
│                            ▼                                    │
│                    ┌───────────────┐                           │
│                    │   HTTP 节点    │                           │
│                    │  (知识检索)    │                           │
│                    └───────┬───────┘                           │
└────────────────────────────┼────────────────────────────────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │   RAG API 服务   │
                    │  POST /search    │
                    └─────────────────┘
```

### 4.2 出题工作流配置

**Dify 出题工作流**
```
输入变量：user_id, question_type, difficulty, topic

节点流程：
1. 开始节点：接收参数
2. HTTP 节点：调用 RAG 检索
   - URL: {{RAG_API_URL}}/search
   - Body: {"query": "{{topic}}", "top_k": 5}
3. LLM 节点：生成题目
   - Prompt: 基于检索内容生成{{question_type}}类型题目
4. 结束节点：输出题目

输出变量：question, options, answer, explanation, sources
```

### 4.3 批阅工作流配置

**Dify 批阅工作流**
```
输入变量：user_id, question, user_answer, correct_answer

节点流程：
1. 开始节点：接收参数
2. HTTP 节点：调用 RAG 检索（获取参考内容）
3. LLM 节点：评判答案
   - Prompt: 对比用户答案与标准答案，给出评分和解析
4. 结束节点：输出批阅结果

输出变量：score, is_correct, feedback, correct_answer, sources
```

### 4.4 API 地址配置

在 Dify 中配置环境变量：

```yaml
# 环境变量
RAG_API_URL: http://your-server:5001

# HTTP 节点配置示例
URL: {{RAG_API_URL}}/search
Method: POST
Headers:
  Content-Type: application/json
Body:
  {
    "query": "{{input_text}}",
    "top_k": 5
  }
```

## 五、知识库管理

### 5.1 文档格式支持

| 格式 | 扩展名 | 解析方式 | 元数据 |
|------|--------|----------|--------|
| PDF | .pdf | pdfplumber | 页码、表格 |
| Word | .docx | python-docx | 段落、表格 |
| Excel | .xlsx | openpyxl | 工作表、行列 |
| TXT | .txt | 直接读取 | 无 |

### 5.2 文档管理命令

```bash
# 构建知识库（首次或完全重建）
python rag_demo.py --rebuild

# 增量同步（添加/删除文档后）
python rag_demo.py --sync

# 查看已索引文档
python rag_demo.py --list

# 构建知识图谱
python graph_build.py

# 测试图谱检索
python graph_test.py
```

## 七、部署指南

### 7.1 Docker 部署 Neo4j

```bash
docker run -d \
  --name neo4j \
  -p 7474:7474 -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/password123 \
  -v neo4j_data:/data \
  neo4j:latest
```

### 7.2 启动 API 服务

```bash
# 激活虚拟环境
source venv/bin/activate  # Linux/Mac

venv\Scripts\activate  # Windows
```
