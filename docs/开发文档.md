## 一、项目概述

### 1.1 项目定位

本项目是智能出题系统的**核心知识服务层**，为上层 Dify 工作流提供知识检索能力。系统通过 RAG（检索增强生成）技术，实现基于企业制度文档的智能问答，支持：

- **知识库问答**：基于向量检索 + BM25 + Rerank 的混合检索
- **图谱推理**：基于 Neo4j 的多跳关系查询
- **网络搜索**：实时信息获取（天气、新闻等）
- **Dify 集成**：HTTP API 供工作流调用

### 1.3 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| API 服务 | Flask + Flask-CORS | RESTful API，支持跨域 |

| 关键词检索 | BM25 + jieba | 中文分词 + 倒排索引 |

| 大模型 | Qwen3.5 (通义千问) | 问答生成、实体提取 |

## 三、API 接口文档

### 3.1 基础信息

- **Base URL**: `http://localhost:5001`
- **Content-Type**: `application/json`
- **认证方式**: 无（内网使用）

### 3.2 核心接口

#### 3.2.1 智能聊天

**POST /chat**

支持网络搜索的智能对话，适合实时信息查询。

**请求体**
```json
{
  "user_id": "user_001",
  "session_id": null,
  "message": "今天南京天气怎么样？"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user_id | string | 是 | 用户唯一标识 |
| session_id | string | 否 | 会话ID，首次传 null |
| message | string | 是 | 用户消息 |

**响应体**
```json
{
  "session_id": "abc123...",
  "answer": "根据天气预报，南京今日气温...",
  "mode": "chat",
  "sources": [
    {
      "source": "https://weather.com/...",
      "type": "网络搜索",
      "snippet": "南京天气预报..."
    }
  ],
  "web_searched": true
}
```

**触发网络搜索的关键词**
- 时间类：今天、今日、最新、当前、现在、最近、近期
- 信息类：天气、新闻、股价、行情、汇率、热点

---

#### 3.2.2 知识库问答

**POST /rag**

基于知识库的精确问答，支持多源检索。

**请求体**
```json
{
  "user_id": "user_001",
  "session_id": null,
  "message": "差旅管理办法由哪个部门负责？"
}
```

**响应体**
```json
{
  "session_id": "abc123...",
  "answer": "根据《差旅管理办法》第三条规定...",
  "mode": "rag",
  "sources": [
    {
      "source": "差旅管理办法.pdf",
      "type": "知识库",
      "snippet": "第三条 本办法由人力资源部负责..."
    }
  ]
}
```

**检索策略**
1. Agent 分析问题，决定检索策略
2. 向量检索 + BM25 混合检索
3. Rerank 重排序
4. 若涉及实体关系，触发图谱检索
5. 若知识库不足，触发网络搜索
6. 多源融合生成答案

---

#### 3.2.3 混合检索（Dify 集成）

**POST /search**

供 Dify 工作流调用的纯检索接口，不调用 LLM。

**请求体**
```json
{
  "query": "出差报销流程",
  "top_k": 5
}
```

**响应体**
```json
{
  "contexts": [
    "出差报销流程规定：1. 填写报销单...",
    "报销需附原始凭证..."
  ],
  "metadatas": [
    {"source": "财务制度.pdf", "page": 15},
    {"source": "财务制度.pdf", "page": 16}
  ],
  "scores": [0.95, 0.89]
}
```

**Dify 配置示例**
```
HTTP 节点配置：
- URL: http://your-server:5001/search
- Method: POST
- Headers: {"Content-Type": "application/json"}
- Body: {"query": "{{用户问题}}", "top_k": 5}
- 输出变量: contexts, metadatas, scores
```

#### 3.2.4 图谱检索

**POST /graph/search**

基于知识图谱的关系推理查询。

**请求体**
```json
{
  "query": "信息技术部负责什么？",
  "top_k": 5,
  "depth": 2
}
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| query | string | 是 | - | 查询文本 |
| top_k | int | 否 | 5 | 向量检索数量 |
| depth | int | 否 | 2 | 图谱搜索深度 |

**响应体**
```json
{
  "answer": "信息技术部负责公司信息系统的建设与维护...",
  "entities": ["信息技术部", "信息系统", "网络安全"],
  "has_graph_context": true,
  "sources": [
    {"source": "信息安全管理制度.txt", "type": "知识库"}
  ],
  "graph_context": "【知识图谱信息】\n实体关系：\n- 信息技术部 负责 信息系统\n- 信息技术部 管理 网络安全..."
}
```

## 四、Dify 工作流集成

### 4.1 集成架构

```
┌─────────────────────────────────────────────────────────────────┐
│                     Dify 工作流                                  │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────────────┐ │
│  │  开始节点   │───▶│  LLM 节点   │───▶│      结束节点       │ │
│  │  (用户问题) │    │  (出题/批阅)│    │    (输出结果)       │ │
│  └─────────────┘    └──────┬──────┘    └─────────────────────┘ │
│                            │                                    │
│                            ▼                                    │
│                    ┌───────────────┐                           │
│                    │   HTTP 节点    │                           │
│                    │  (知识检索)    │                           │
│                    └───────┬───────┘                           │
└────────────────────────────┼────────────────────────────────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │   RAG API 服务   │
                    │  POST /search    │
                    └─────────────────┘
```

### 4.2 出题工作流配置

**Dify 出题工作流**
```
输入变量：user_id, question_type, difficulty, topic

节点流程：
1. 开始节点：接收参数
2. HTTP 节点：调用 RAG 检索
   - URL: {{RAG_API_URL}}/search
   - Body: {"query": "{{topic}}", "top_k": 5}
3. LLM 节点：生成题目
   - Prompt: 基于检索内容生成{{question_type}}类型题目
4. 结束节点：输出题目

输出变量：question, options, answer, explanation, sources
```

### 4.3 批阅工作流配置

**Dify 批阅工作流**
```
输入变量：user_id, question, user_answer, correct_answer

节点流程：
1. 开始节点：接收参数
2. HTTP 节点：调用 RAG 检索（获取参考内容）
3. LLM 节点：评判答案
   - Prompt: 对比用户答案与标准答案，给出评分和解析
4. 结束节点：输出批阅结果

输出变量：score, is_correct, feedback, correct_answer, sources
```

### 4.4 API 地址配置

在 Dify 中配置环境变量：

```yaml
# 环境变量
RAG_API_URL: http://your-server:5001

# HTTP 节点配置示例
URL: {{RAG_API_URL}}/search
Method: POST
Headers:
  Content-Type: application/json
Body:
  {
    "query": "{{input_text}}",
    "top_k": 5
  }
```

```bash
python rag_demo.py --sync

# 查看已索引文档
python rag_demo.py --list

# 构建知识图谱
python graph_build.py

# 测试图谱检索
python graph_test.py
```
