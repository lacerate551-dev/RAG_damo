# 数据库设计文档

本文档描述 RAG 知识库系统中所有数据库的结构和用途，包括 SQLite 关系数据库和 ChromaDB 向量数据库。

---

## 数据库文件总览

### SQLite 关系数据库

| 文件名 | 大小 | 主要功能 | 所属模块 |
|--------|------|----------|----------|
| `sessions.db` | ~200KB | 用户认证、会话管理、对话历史、审计日志 | 核心（多模块共用） |
| `sync_data.db` | ~56KB | 知识库同步、文档哈希、变更日志、订阅通知 | `knowledge_sync.py` |
| `feedback.db` | ~44KB | 用户反馈、FAQ建议、质量报告 | `feedback_service.py` |
| `outline_cache.db` | ~204KB | 文档纲要缓存、向量计算结果 | `outline_generator.py` |
| `exam_analysis.db` | ~76KB | 题库分析、知识点关联、整卷分析 | `exam_analysis.py` |

### ChromaDB 向量数据库

| 目录名 | 主要功能 | 所属模块 |
|--------|----------|----------|
| `chroma_db/` | 多向量库存储 | `knowledge_base_manager.py` |
| `bm25_indexes/` | BM25 关键词索引 | `knowledge_base_manager.py` |

## 1. 多向量库架构（ChromaDB）

**所属模块**：`knowledge_base_manager.py`

### 1.1 向量库结构

```
chroma_db/
├── chroma.sqlite3          # ChromaDB 主数据库
├── kb_metadata.json        # 向量库元数据
├── public_kb/              # 公开知识库
├── dept_finance/           # 财务部知识库
├── dept_hr/                # 人事部知识库
├── dept_tech/              # 技术部知识库
├── dept_admin/             # 行政部知识库
├── dept_operation/         # 运营部知识库
├── dept_legal/             # 法务部知识库
├── dept_strategy/          # 战略部知识库
└── dept_general/           # 通用知识库
```

### 1.2 权限矩阵

| 角色 | 可访问向量库 | 可上传 | 可删除 | 可同步 |
|------|------------|--------|--------|--------|
| admin | 全部 | 全部 | 全部 | 全部 |
| manager | public_kb + 本部门 | 本部门 | 本部门 | 本部门 |
| user | public_kb + 本部门 | - | - | - |

### 1.3 文档元数据结构

每个文档 chunk 的元数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| `source` | TEXT | 文档来源文件名 |
| `page` | INTEGER | PDF 页码（可选） |
| `sheet` | TEXT | Excel 工作表（可选） |
| `row` | INTEGER | Excel 行号（可选） |
| `section` | TEXT | 章节（可选） |
| `is_table` | BOOLEAN | 是否为表格 |
| `is_excel` | BOOLEAN | 是否为 Excel 数据 |
| `security_level` | TEXT | 原安全级别（兼容旧数据） |
| `collection` | TEXT | 所属向量库名称 |

### 1.4 BM25 索引

```
bm25_indexes/
├── public_kb.pkl           # 公开知识库 BM25 索引
├── dept_finance.pkl        # 财务部 BM25 索引
├── dept_hr.pkl             # 人事部 BM25 索引
├── dept_tech.pkl           # 技术部 BM25 索引
├── dept_admin.pkl          # 行政部 BM25 索引
├── dept_operation.pkl      # 运营部 BM25 索引
├── dept_legal.pkl          # 法务部 BM25 索引
├── dept_strategy.pkl       # 战略部 BM25 索引
└── dept_general.pkl        # 通用知识库 BM25 索引
```

### 1.5 知识库路由

**所属模块**：`kb_router.py`

路由逻辑：
1. **规则匹配**：根据关键词判断查询意图
2. **LLM 分析**：复杂查询使用 LLM 分析意图
3. **权限过滤**：根据用户角色和部门过滤可访问向量库
4. **结果合并**：多向量库检索结果使用 RRF 融合

部门关键词映射示例：
```python
DEPARTMENT_KEYWORDS = {
    'finance': ['财务', '报销', '发票', '预算', '财务报表'],
    'hr': ['人事', '招聘', '入职', '离职', '考勤', '请假'],
    'tech': ['技术', '开发', '系统', '服务器', '数据库'],
    'admin': ['行政', '办公室', '采购', '董事会'],
    # ...
}
```

## 2. sessions.db - 核心数据库

**共用模块**：`auth_gateway.py`、`session_manager.py`、`audit_logger.py`

### 2.1 users 表 - 用户表

| 字段 | 类型 | 说明 |
|------|------|------|
| `user_id` | TEXT | 用户ID（UUID），主键 |
| `username` | TEXT | 用户名，唯一 |
| `password_hash` | TEXT | 密码哈希（bcrypt） |
| `role` | TEXT | 角色：admin / manager / user |
| `department` | TEXT | 部门 |
| `is_active` | INTEGER | 是否激活（0/1） |
| `created_at` | TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMP | 更新时间 |

### 2.2 role_permissions 表 - 角色权限表

| 字段 | 类型 | 说明 |
|------|------|------|
| `role` | TEXT | 角色名称，主键 |
| `allowed_levels` | TEXT | 允许访问的安全级别（JSON数组） |

### 2.3 sessions 表 - 会话表

| 字段 | 类型 | 说明 |
|------|------|------|
| `session_id` | TEXT | 会话ID（UUID），主键 |
| `user_id` | TEXT | 所属用户ID |
| `created_at` | TIMESTAMP | 创建时间 |
| `last_active` | TIMESTAMP | 最后活跃时间 |
| `metadata` | TEXT | 元数据（JSON格式） |

**作用**：实现多用户会话隔离，支持多轮对话记忆。

### 2.4 messages 表 - 消息历史表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER | 自增主键 |
| `session_id` | TEXT | 关联会话ID |
| `role` | TEXT | 角色：user / assistant |
| `content` | TEXT | 消息内容 |
| `created_at` | TIMESTAMP | 创建时间 |

**索引**：
- `idx_messages_session(session_id, created_at)` - 加速会话历史查询

### 2.5 audit_logs 表 - 审计日志表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER | 自增主键 |
| `user_id` | TEXT | 用户ID |
| `username` | TEXT | 用户名 |
| `action` | TEXT | 操作类型（chat/rag/search等） |
| `query` | TEXT | 用户查询内容 |
| `result_summary` | TEXT | 结果摘要 |
| `sources` | TEXT | 来源文档（JSON数组） |
| `role` | TEXT | 用户角色 |
| `department` | TEXT | 用户部门 |
| `ip_address` | TEXT | 客户端IP |
| `duration_ms` | INTEGER | 处理耗时（毫秒） |
| `created_at` | TIMESTAMP | 创建时间 |

**作用**：记录所有用户操作，用于安全审计和行为分析。

## 3. sync_data.db - 知识库同步数据库

**所属模块**：`knowledge_sync.py`

### 3.1 document_hashes 表 - 文档哈希表

| 字段 | 类型 | 说明 |
|------|------|------|
| `document_id` | TEXT | 文档ID（相对路径），主键 |
| `document_name` | TEXT | 文件名 |
| `content_hash` | TEXT | 文档内容MD5哈希 |
| `file_size` | INTEGER | 文件大小（字节） |
| `last_modified` | TIMESTAMP | 最后修改时间 |
| `created_at` | TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMP | 更新时间 |

**作用**：记录每个文档的当前状态，用于检测变更。

### 3.2 change_logs 表 - 变更日志表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER | 自增主键 |
| `document_id` | TEXT | 文档ID |
| `document_name` | TEXT | 文件名 |
| `change_type` | TEXT | 变更类型：added / modified / deleted |
| `old_hash` | TEXT | 变更前哈希 |
| `new_hash` | TEXT | 变更后哈希 |
| `change_time` | TIMESTAMP | 变更时间 |
| `processed` | INTEGER | 是否已处理（0/1） |
| `error_message` | TEXT | 错误信息 |
| `created_at` | TIMESTAMP | 创建时间 |

**索引**：
- `idx_change_logs_time(change_time)` - 按时间查询
- `idx_change_logs_processed(processed)` - 查询未处理记录

### 3.3 subscriptions 表 - 用户订阅表

### 3.4 notifications 表 - 通知记录表

## 数据库维护

### 数据清理

```bash
# 清理过期会话（24小时未活跃）
sqlite3 sessions.db "DELETE FROM sessions WHERE last_active < datetime('now', '-24 hours');"
sqlite3 sessions.db "DELETE FROM messages WHERE session_id NOT IN (SELECT session_id FROM sessions);"

# 清理旧审计日志（保留30天）
sqlite3 sessions.db "DELETE FROM audit_logs WHERE created_at < datetime('now', '-30 days');"

# 清理已处理的通知（保留7天）
sqlite3 sync_data.db "DELETE FROM notifications WHERE read = 1 AND created_at < datetime('now', '-7 days');"
```

sqlite3 sessions.db ".backup backup/sessions_backup.db"

### 重置数据库

删除对应的文件，服务启动时会自动重建表结构：

```bash
# 重置会话数据（会丢失用户和对话历史）
rm sessions.db

# 重置向量库（会丢失所有知识库内容）
rm -rf chroma_db/*
rm -rf bm25_indexes/*

# 重置同步数据（会丢失文档追踪和订阅）
rm sync_data.db

# 重置反馈数据
rm feedback.db

# 重置纲要缓存
rm outline_cache.db

# 重置题库分析数据
rm exam_analysis.db
```
