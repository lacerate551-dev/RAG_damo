## 模块详细说明

### 一、核心 RAG 模块

#### 1. `rag_demo.py` - RAG 基础功能
**职责**：知识库基础功能的入口点

**主要功能**：
- 文档解析（PDF、Word、Excel、TXT）
- 向量索引构建（ChromaDB）
- BM25 关键词索引
- 混合检索（向量 + BM25 + Rerank）
- 知识库问答

**关键函数**：
| 函数 | 说明 |
|------|------|
| `load_documents()` | 加载文档目录 |
| `build_vector_index()` | 构建向量索引 |
| `build_bm25_index()` | 构建 BM25 索引 |
| `search_knowledge()` | 混合检索 |
| `generate_answer()` | LLM 生成答案 |

**依赖关系**：
- 被 `agentic_rag.py`、`graph_rag.py` 调用
- 依赖：ChromaDB、BGE 向量模型

**是否需要优化**：⚠️ 可优化
- 考虑拆分为更细粒度的模块（文档处理、索引构建、检索）
- 添加异步支持

---

#### 2. `agentic_rag.py` - Agentic RAG 核心
**职责**：智能问答的 Agent 决策引擎

**主要功能**：
- Agent 决策循环（检索、改写、分解、回答）
- 网络搜索集成（Serper API）
- 图谱检索集成
- 多源结果融合
- 查询重写与问题分解

**关键类/函数**：
| 函数 | 说明 |
|------|------|
| `AgenticRAG` | 主类，封装所有 Agent 功能 |
| `process()` | 处理用户查询 |
| `_decide_action()` | Agent 决策下一步操作 |
| `_web_search()` | 网络搜索 |
| `_rewrite_query()` | 查询重写 |

**依赖关系**：
- 依赖：`rag_demo.py`、`graph_rag.py`（可选）
- 被 `rag_api_server.py` 调用

**是否需要优化**：⚠️ 可优化
- Agent 决策逻辑可配置化
- 添加更多工具支持

---

### 二、图谱模块

#### 3. `graph_manager.py` - 图谱管理器
**职责**：Neo4j 图数据库管理

**主要功能**：
- Neo4j 连接管理
- 实体 CRUD 操作
- 关系 CRUD 操作
- 子图检索
- 图谱统计

**关键类/函数**：
| 类/函数 | 说明 |
|---------|------|
| `GraphManager` | 图谱管理主类 |
| `Entity` | 实体数据类 |
| `Triple` | 三元组数据类 |
| `get_graph_manager()` | 单例获取 |

**依赖关系**：
- 被 `graph_rag.py`、`graph_build.py` 调用
- 依赖：neo4j 驱动

**是否需要优化**：✅ 结构合理
- 考虑添加连接池

---

#### 4. `entity_extractor.py` - 实体提取器
**职责**：使用 LLM 从文本提取实体和关系

**主要功能**：
- 文本实体识别（部门、制度、人员、流程）
- 关系三元组提取
- 批量处理支持

**依赖关系**：
- 被 `graph_build.py` 调用
- 依赖：OpenAI API / 通义千问 API

**是否需要优化**：✅ 结构合理

---

#### 5. `graph_rag.py` - 图谱 RAG
**职责**：图谱增强检索

**主要功能**：
- 向量检索 + 图谱检索融合
- 多跳推理
- 实体感知检索

**依赖关系**：
- 依赖：`rag_demo.py`、`graph_manager.py`、`entity_extractor.py`
- 被 `agentic_rag.py` 调用

**是否需要优化**：✅ 结构合理

---

#### 6. `graph_build.py` - 图谱构建工具
**职责**：命令行图谱构建工具

**使用方式**：
```bash
python graph_build.py --stats                    # 查看统计
python graph_build.py --file documents/xxx.pdf   # 构建单文件
python graph_build.py --dir documents/人事制度    # 构建目录
python graph_build.py --clear --file xxx.pdf     # 清空重建

```

**是否需要优化**：✅ 功能完整

### 三、出题系统模块

#### 7. `exam_manager.py` - 出题核心逻辑
**职责**：试卷生成、保存、批阅的核心业务逻辑

**主要功能**：
- 调用 Dify 工作流生成试卷
- 试卷 CRUD 操作
- 审核流程管理
- 自动批阅与报告生成

**关键函数**：
| 函数 | 说明 |
|------|------|
| `generate_exam()` | 生成试卷 |
| `save_exam()` | 保存试卷到题库 |
| `grade_exam()` | 批阅试卷 |
| `review_exam()` | 审核试卷 |

**数据存储**：
- 试卷：`./题库/` 目录（JSON 文件）
- 报告：`./批阅报告/` 目录（JSON 文件）

**是否需要优化**：⚠️ 可优化
- 考虑使用 SQLite 替代文件存储
- 添加试卷版本管理

#### 8. `exam_api.py` - 出题系统 API

**API 端点**：

**依赖关系**：
- 依赖：`exam_manager.py`、`auth.py`

**是否需要优化**：✅ 结构合理

### 四、安全与认证模块

#### 10. `auth.py` - 用户认证
**职责**：JWT 认证与角色管理

**主要功能**：
- 用户注册/登录
- JWT Token 签发与验证
- 角色权限控制（RBAC）
- `@require_auth` 装饰器

**角色体系**：
| 角色 | 权限 |
|------|------|
| admin | 所有文档 + 用户管理 |
| manager | public + internal + confidential |
| user | public + internal |

**关键装饰器**：
```python
@require_auth           # 需要登录
@require_admin          # 需要管理员
```

**数据存储**：SQLite（`./sessions.db`）

**是否需要优化**：✅ 结构合理

---

#### 11. `security.py` - 安全防护

- 输出过滤（敏感信息过滤）

## 优化建议总结

### 高优先级

| 模块 | 问题 | 建议 |
|------|------|------|
| `rag_demo.py` | 功能过多，职责不清 | 拆分为：`document_processor.py`、`index_builder.py`、`retriever.py` |
| `rag_api_server.py` | 文件过大 | 按功能拆分为多个 Blueprint |
| `exam_manager.py` | 文件存储不便管理 | 改用 SQLite 存储 |

### 中优先级

| 模块 | 问题 | 建议 |
|------|------|------|
| `agentic_rag.py` | Agent 决策逻辑硬编码 | 配置化，支持自定义工具 |
| `session_manager.py` | SQLite 单机限制 | 添加 Redis 支持 |

### 低优先级

| 模块 | 问题 | 建议 |
|------|------|------|
| `graph_manager.py` | 无连接池 | 添加连接池优化 |
| `config.py` | 敏感信息暴露风险 | 使用环境变量或密钥管理服务 |
