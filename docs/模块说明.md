## 模块详细说明

### 一、核心 RAG 模块

#### 1. `rag_demo.py` - RAG 基础功能
**职责**：知识库基础功能的入口点

**主要功能**：
- 文档解析（PDF、Word、Excel、TXT）
- 向量索引构建（ChromaDB）
- BM25 关键词索引
- 混合检索（向量 + BM25 + Rerank）
- 知识库问答
- **多向量库支持**（public_kb + dept_xxx）

**关键函数**：
| 函数 | 说明 |
|------|------|
| `load_documents()` | 加载文档目录 |
| `build_knowledge_base()` | 构建知识库（单向量库模式） |
| `search_knowledge()` | 混合检索（支持多向量库） |
| `_search_multi_kb()` | 多向量库并行检索 |
| `generate_answer()` | LLM 生成答案 |

**多向量库模式**：
```python
# 启用多向量库模式
USE_MULTI_KB = True

# 多向量库检索
results = search_knowledge(
    query="财务报表",
    top_k=5,
    role="manager",          # 用户角色
    department="finance"     # 用户部门
)
```

**依赖关系**：
- 被 `agentic_rag.py`、`graph_rag.py`、`knowledge_sync.py` 调用
- 依赖：ChromaDB、BGE 向量模型、`knowledge_base_manager.py`

---

#### 2. `knowledge_base_manager.py` - 多向量库管理器
**职责**：多向量库的创建、管理和检索

**主要功能**：
- 多向量库创建与管理（public_kb + dept_xxx）
- 每个向量库独立的 BM25 索引
- 并行检索多个向量库
- RRF（Reciprocal Rank Fusion）结果融合
- 向量库元数据管理

**关键类/函数**：
| 类/函数 | 说明 |
|---------|------|
| `KnowledgeBaseManager` | 多向量库管理主类 |
| `BM25Index` | BM25 索引管理类 |
| `CollectionInfo` | 向量库信息数据类 |
| `SearchResult` | 检索结果数据类 |
| `get_kb_manager()` | 单例获取 |

**使用方式**：
```python
from knowledge_base_manager import KnowledgeBaseManager

kb_manager = KnowledgeBaseManager('./vector_store/chroma')

# 创建向量库
kb_manager.create_collection('dept_finance', display_name='财务部知识库')

# 获取向量库列表
collections = kb_manager.list_collections()

# 检索
results = kb_manager.search_single('public_kb', query_vector, top_k=5)

# 并行检索多个库
results = kb_manager.search_multiple(['public_kb', 'dept_finance'], query_vector)
```

**依赖关系**：
- 被 `rag_demo.py`、`rag_api_server.py` 调用
- 依赖：ChromaDB、rank_bm25、jieba

---

#### 3. `kb_router.py` - 知识库路由器
**职责**：根据查询意图和用户权限智能选择目标向量库

**主要功能**：
- 规则匹配（关键词识别部门）
- LLM 意图分析（复杂查询）
- 权限过滤
- 目标向量库选择

**关键类/函数**：
| 类/函数 | 说明 |
|---------|------|
| `KnowledgeBaseRouter` | 知识库路由主类 |
| `QueryIntent` | 查询意图数据类 |
| `route_query()` | 便捷路由函数 |

**部门关键词映射**：
```python
DEPARTMENT_KEYWORDS = {
    'finance': ['财务', '报销', '发票', '预算', '财务报表'],
    'hr': ['人事', '招聘', '入职', '离职', '考勤', '请假'],
    'tech': ['技术', '开发', '系统', '服务器', '数据库'],
    'admin': ['行政', '办公室', '采购', '董事会'],
    # ...
}
```

**使用方式**：
```python
from kb_router import route_query

targets = route_query(
    query="财务部的报销流程",
    role="manager",
    department="finance"
)
# 返回: ['dept_finance', 'public_kb']
```

#### 4. `agentic_rag.py` - Agentic RAG 核心
**职责**：智能问答的 Agent 决策引擎

- 网络搜索集成（Serper API）

| `process()` | 处理用户查询 |

| `_decide_action()` | Agent 决策下一步操作 |

**依赖关系**：

### 二、图谱模块（graph/）

#### 5. `graph/graph_manager.py` - 图谱管理器
**职责**：Neo4j 图数据库管理

**主要功能**：
- Neo4j 连接管理
- 实体 CRUD 操作
- 关系 CRUD 操作
- 子图检索
- 图谱统计

**关键类/函数**：
| 类/函数 | 说明 |
|---------|------|
| `GraphManager` | 图谱管理主类 |
| `Entity` | 实体数据类 |
| `Triple` | 三元组数据类 |
| `get_graph_manager()` | 单例获取 |

**使用方式**：
```python
from graph import GraphManager, Entity, Triple

gm = GraphManager()
gm.connect()
gm.create_entity("人力资源部", "部门", {"描述": "负责公司人事管理"})
gm.create_relation("人力资源部", "负责", "差旅管理办法")
```

#### 6. `graph/entity_extractor.py` - 实体提取器

- 批量处理支持

**使用方式**：
```python
from graph import EntityExtractor

extractor = EntityExtractor()
triples = extractor.extract("差旅管理办法由人力资源部负责...")
```

#### 7. `graph/graph_rag.py` - 图谱 RAG
**职责**：图谱增强检索

```python
from graph import GraphRAG

rag = GraphRAG()
```

### 三、出题系统模块

#### 9. `exam_manager.py` - 出题核心逻辑
**职责**：试卷生成、保存、批阅的核心业务逻辑

```
生成试卷 → draft (草稿)
     ↓
提交审核 → pending_review (待审核)
     ↓
管理员审核 → approved (通过) / rejected (驳回)
     ↓
学生答题 → 批阅 → 生成报告
```

**关键函数**：
| 函数 | 说明 |
|------|------|
| `generate_exam_by_file()` | 按文件生成题目（带溯源） |
| `grade_from_mysql()` | 基于 MySQL 数据批卷 |
| `grade_question()` | 单题批阅（调用 Dify 工作流） |

### 四、安全与认证模块

#### 14. `auth_gateway.py` - 网关认证
**职责**：网关注入的 Header 认证与多向量库权限控制

**主要功能**：
- 从网关注入的 Header 读取用户信息
- 角色映射（后端角色 → 本地角色）
- 多向量库权限控制
- `@require_gateway_auth` 装饰器
- `@require_role` 装饰器

**网关注入的 Header**：
| Header | 说明 |
|--------|------|
| `X-User-ID` | 用户唯一标识 |
| `X-User-Name` | 用户名 |
| `X-User-Role` | 用户角色 |
| `X-User-Department` | 部门 |

**多向量库权限配置**：
```python
COLLECTION_PERMISSIONS = {
    'admin': {
        'read': ['*'],           # 可读取所有向量库
        'write': ['*'],          # 可写入所有向量库
        'create': True,          # 可创建新向量库
        'drop': True,            # 可删除向量库
    },
    'manager': {
        'read': ['public_kb', 'dept_{dept}'],
        'write': ['dept_{dept}'],
        'create': False,
    },
    'user': {
        'read': ['public_kb', 'dept_{dept}'],
        'write': [],
        'create': False,
    }
}
```

**关键函数**：
| 函数 | 说明 |
|------|------|
| `get_accessible_collections()` | 获取用户可访问的向量库 |
| `check_collection_permission()` | 检查向量库操作权限 |
| `get_user_permissions()` | 获取用户安全级别权限 |
